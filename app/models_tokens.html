How the workflow runs

ImageCluster encodes every image in the img folder into a vector, reduces those vectors to 2D, and lets you explore the result on an interactive canvas with optional clustering and text search.

1
Scan the img folder
The backend lists supported images and reports the count.
2
Pick an embedding model
CLIP-family models support text search; vision-only models support projection and clustering only.
3
Generate the graph
Runs UMAP or t-SNE plus optional K-Means clustering as a background job.
4
Explore
Pan and zoom the canvas, switch to image mode, search semantically, and browse clusters in the gallery.

Choosing a model

Match the model to the collection and hardware.

  • General art collection, need text search: start with OpenCLIP ViT-B/32 or SigLIP 2.
  • Low-power machine or very large folder: MobileCLIP variants.
  • Formal visual similarity, no text: DINOv2.
  • Multilingual or non-Western corpus: MetaCLIP 2 worldwide.
Models marked planned are roadmap entries and cannot run locally yet.

Embedding model guide

Embedding
A numeric vector that represents an image or text in a shared space.
Projection
A 2D map of high-dimensional embeddings, produced by UMAP or t-SNE.
UMAP
Fast projection method that preserves useful local and global structure.
t-SNE
Projection method that emphasizes tight local neighborhoods.
Clustering (K-Means)
Groups embeddings into K clusters and drives plot colors and gallery groups.
Semantic search
Free-text query matched against image embeddings, available for compatible image-text models.
Searchable index
A prepared structure over embeddings that makes repeated text queries faster.
Score / threshold
Similarity score per result; thresholds hide matches below a cutoff.
Top results
Maximum number of matches returned by a search.
Hugging Face token
A local credential used by model loaders to download gated models.
